2. Variables, interpolation, incrémentation

Affectation et conventions

score = 0            # = AFFECTE (== compare)
prenom = "Zoé"       # snake_case pour les variables
TVA = 0.20           # MAJUSCULES = constante

Incrémentation (pas de ++ en Ruby)

score = score + 1    # version longue
score += 1           # raccourci idiomatique
score -= 2           # existe aussi en -, *, /

Interpolation (guillemets DOUBLES)

age = 30
puts "j'ai #{age} ans"       # -> j'ai 30 ans
puts 'j\'ai #{age} ans'      # simples : PAS interprété
puts "dans 10 ans : #{age + 10}" # toute expression marche
💡 Piège vécu en S1 : les guillemets courbes “ ” collés depuis un doc externe cassent la syntaxe. Toujours des guillemets droits ".

3. Conditions

if / elsif / else

if note >= 16
  puts "mention bien"
elsif note >= 10       # elsif, JAMAIS elseif
  puts "admis"
else
  puts "rattrapage"
end

Sur une ligne + unless

puts "gagné" if score > 100
puts "silence" unless volume > 0 # unless = if not

case / when

case meteo
when "soleil"
  puts "terrasse"
when "pluie", "orage"    # plusieurs valeurs par when
  puts "capuche"
else
  puts "on verra"
end
⚠️ L'ordre des branches compte : le cas PARTICULIER se teste avant le cas général (piège du jour 3).

5. Méthodes

# def + snake_case ; parenthèses pour les paramètres
def prix_ttc(prix_ht, taux = 0.20)  # taux a une valeur PAR DÉFAUT
  prix_ht * (1 + taux)   # dernière expression évaluée = RETOUR IMPLICITE
end

puts prix_ttc(100)        # -> 120.0 (le défaut s'applique)
puts prix_ttc(100, 0.055) # -> 105.5 (argument fourni)
Le trio à ne plus confondre :
· puts AFFICHE pour l'humain et renvoie nil
· return TRANSMET une valeur au code appelant
· sans return, Ruby renvoie la dernière expression évaluée
Un test lit la VALEUR DE RETOUR, jamais l'écran.
Argument vs paramètre : le paramètre est le nom déclaré dans le def ; l'argument est la valeur passée à l'appel.
