🧠 Mes questions & incompréhensions

« J'écris each_slice sur le nom de ma méthode et ça plante. »
Confusion nom de méthode / nom de paramètre. def sorted_pairs(an_array) : la méthode s'appelle sorted_pairs, mais la donnée sur laquelle on travaille s'appelle an_array. Une méthode ne se parcourt pas : c'est le TABLEAU reçu en paramètre qu'on itère. Réflexe : « sur quel objet je veux agir ? » et pas « comment s'appelle mon exercice ? ».
« Pourquoi each ne me renvoie pas le résultat que je construis ? »
Parce que each renvoie le tableau de DÉPART, pas ce qui se passe dedans. Son seul rôle est de parcourir. Si tu veux récupérer une transformation, il te faut soit map (qui renvoie un nouveau tableau des valeurs du bloc), soit un accumulateur déclaré AVANT la boucle et renvoyé APRÈS.
« Ma variable déclarée dans le bloc n'existe plus après. »
Un bloc est un contenant. Une variable créée à l'intérieur meurt à la fermeture du bloc. Une variable créée AVANT le bloc est visible à l'intérieur, et les modifications qu'on lui fait survivent. D'où le pattern de l'accumulateur : somme = 0 avant, somme += ... dedans, somme après.
« Dans un select, je mets un if ? »
Non. select attend déjà un booléen : il garde l'élément si le bloc est vrai. Écrire if number.even? dedans, c'est faire deux fois le travail. AS IS select { |n| if n.even? } → TO BE select { |n| n.even? }.
« { } ou do...end ? Et je mets les |variables| où ? »
Les deux syntaxes font la même chose. Convention : { } sur une seule ligne, do...end sur plusieurs. Dans les deux cas, les paramètres du bloc sont DANS le bloc, juste après l'ouverture : { |n| ... } ou do |n| ... end. Jamais avant l'accolade, jamais après end.
« count ne fait pas la même chose selon les cas ? »
Exact. Sur un Array, count sans argument renvoie le nombre d'éléments. Sur une String, count EXIGE un argument : c'est le nombre d'occurrences des caractères donnés. "hello".count plante ; "hello".count("l") renvoie 2. Pour la longueur d'une chaîne, c'est length ou size.
« Comment j'imbrique des guillemets dans une interpolation ? »
Dans une chaîne délimitée par ", un guillemet double interne doit être échappé avec un antislash : \". Et on n'a pas besoin de concaténer avec + si on interpole : "<a href=\"#{url}\">" est complet, tout est dans une seule chaîne.
« Argument ou attribut ? »
Un ARGUMENT est une valeur qu'on passe à une méthode au moment de l'appel : il vit le temps de l'appel. Un ATTRIBUT est une donnée qui appartient à l'objet et qui persiste entre les appels. player.attack(10) : 10 est un argument, player.health est un attribut.

🎯 Concepts clés

Le bloc, un morceau de code passé en argument
Un itérateur ne sait pas quoi faire des éléments : c'est le bloc qui le lui dit. select sait filtrer, mais c'est TON bloc qui définit le critère. C'est pour ça qu'un seul itérateur sert à mille usages.
Choisir son itérateur par sa VALEUR DE RETOUR
C'est le seul critère qui compte. each renvoie le tableau d'origine (donc on l'utilise pour ses effets de bord). map renvoie un tableau transformé, même taille. select/reject renvoient un tableau filtré, taille plus petite. find renvoie UN élément. reduce renvoie une seule valeur agrégée.
Le pattern accumulateur
Déclarer la variable AVANT la boucle (sinon elle meurt à chaque tour et n'existe plus après), la modifier DANS la boucle, la renvoyer APRÈS. C'est la mécanique manuelle de ce que reduce automatise.
yield : ma méthode accepte un bloc
yield exécute le bloc passé à l'appel, à l'endroit exact où il est écrit. C'est le mécanisme qui permet de laisser l'appelant décider d'une partie du comportement, sans que la méthode connaisse ce comportement à l'avance.
Paramètre optionnel
def tag(name, content = nil) : si l'appelant n'a rien passé, le paramètre vaut nil, et on peut brancher dessus. Cela permet une méthode à deux comportements sans en écrire deux.

⚠️ Pièges du jour

Erreur : undefined method 'each_slice' for nil ou NoMethodError sur le nom de la méthode.
Cause : l'itérateur est appliqué au nom de la méthode et non au paramètre.
Correctif : AS IS sorted_pairs.each_slice(2) → TO BE an_array.each_slice(2) # on itère sur la DONNÉE, pas sur le nom de l'exercice
Erreur : la méthode renvoie le tableau de départ, non transformé.
Cause : each renvoie toujours son receveur. Il ne collecte rien.
Correctif : AS IS an_array.each { |n| n * 2 } → TO BE an_array.map { |n| n * 2 } # map collecte les valeurs du bloc
Erreur : select renvoie tous les éléments au lieu de filtrer.
Cause : if redondant à l'intérieur du bloc, dont la valeur de retour n'est plus le booléen attendu.
Correctif : AS IS select { |n| if n.even? } → TO BE select { |n| n.even? } # le bloc DOIT valoir true ou false
Erreur : ArgumentError: wrong number of arguments sur "hello".count.
Cause : sur une String, count exige un argument. Ce n'est pas la même méthode que sur Array.
Correctif : AS IS word.count → TO BE word.size # pour compter les caractères d'une chaîne
Erreur : la variable construite dans le bloc est nil après la boucle.
Cause : elle a été déclarée À L'INTÉRIEUR du bloc, elle meurt à chaque fermeture.
Correctif : déclarer l'accumulateur AVANT la boucle. # il survit alors à chaque itération

🔧 Patterns & idiomes

def sum_odd_indexed(an_array)                    # le paramètre s'appelle an_array, PAS sum_odd_indexed
  somme = 0                                      # accumulateur déclaré AVANT la boucle : il doit lui survivre
  an_array.each_with_index do |element, index|   # each_with_index fournit DEUX variables de bloc : la valeur et sa position
    somme += element if index.odd?               # on n'ajoute que si l'index est impair (if en modificateur, en fin de ligne)
  end                                            # fin du bloc : each_with_index renvoie an_array, on l'ignore
  somme                                          # c'est l'accumulateur qu'on renvoie, pas le retour de each
end
def even_numbers(an_array)                       # objectif : garder uniquement les nombres pairs
  an_array.select { |number| number.even? }      # select garde l'élément si le bloc vaut true : pas de if à l'intérieur
end                                              # select renvoie un NOUVEAU tableau filtré, retour implicite
def short_words(an_array)                        # objectif : retirer les mots longs
  an_array.reject { |word| word.size > 5 }       # reject est l'inverse de select : il ÉLIMINE si le bloc vaut true
end                                              # on utilise size (et non count) sur une String
def first_under(an_array, limit)                 # objectif : le PREMIER élément sous la limite
  an_array.find { |number| number < limit }      # find s'arrête au premier match et renvoie l'ÉLÉMENT (pas un tableau)
end                                              # renvoie nil si aucun élément ne correspond
def concatenate(an_array)                        # objectif : coller toutes les chaînes bout à bout
  an_array.reduce("") { |resultat, mot| resultat + mot } # "" est l'accumulateur initial ; à chaque tour on lui ajoute le mot
end                                              # reduce renvoie l'accumulateur final : UNE seule valeur
def sorted_pairs(an_array)                       # objectif : trier chaque paire d'éléments consécutifs
  an_array.each_slice(2)                         # découpe le tableau en tranches de 2 éléments
          .map { |paire| paire.sort }            # map transforme chaque tranche en sa version triée
end                                              # chaînage : la sortie de each_slice devient l'entrée de map
def tag(name, content = nil)                     # content est optionnel : vaut nil si non fourni
  if content.nil?                                # si aucun contenu n'a été passé...
    "<#{name}>#{yield}</#{name}>"                # ...on exécute le BLOC passé à l'appel avec yield
  else                                           # sinon...
    "<#{name}>#{content}</#{name}>"              # ...on utilise directement le contenu fourni
  end                                            # la branche empruntée est la valeur de retour
end

tag("p") { "Bonjour" }                           # le bloc est ce que yield ira exécuter => "<p>Bonjour</p>"

❓ Auto-test (réponds de mémoire, puis déplie)

Que renvoie [1, 2, 3].each { |n| n * 2 } ?
[1, 2, 3]. each renvoie toujours son receveur. Pour [2, 4, 6], il fallait map.
Pourquoi ne faut-il PAS écrire select { |n| if n.even? } ?
Parce que select attend déjà un booléen. Le if est redondant et casse la valeur de retour du bloc. Il faut select { |n| n.even? }.
Où déclarer un accumulateur, et pourquoi ?
AVANT la boucle. Une variable créée dans le bloc meurt à la fermeture du bloc, elle ne survit ni aux tours suivants ni à l'après-boucle.
"hello".count : qu'est-ce qui se passe ?
ArgumentError. Sur une String, count exige un argument. Pour la longueur, c'est size ou length.
Quelle différence entre un argument et un attribut ?
L'argument est passé à la méthode au moment de l'appel et vit le temps de l'appel. L'attribut appartient à l'objet et persiste entre les appels.
À quoi sert yield ?
À exécuter, à l'endroit où il est écrit, le bloc passé à la méthode par l'appelant.
