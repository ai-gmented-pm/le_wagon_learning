🧠 Mes questions & incompréhensions

« upcase et capitalize, lequel fait quoi ? » (je les avais inversés)
upcase met TOUT en majuscules. capitalize ne touche que la première lettre et force le reste en minuscules. Le nom est trompeur : capitalize ne veut pas dire « tout en capitales », il veut dire « appliquer la capitale initiale ».
« Comment je mets une majuscule à chaque mot d'une phrase ? »
Ruby n'a pas de méthode native pour ça. On compose : découper la phrase en mots, capitaliser chaque mot, recoller. Soit "ma phrase".split.map(&:capitalize).join(" "). Attention : titleize existe uniquement dans Rails (ActiveSupport). En Ruby seul, il provoque un NoMethodError, ce que j'ai constaté dans pry.
« Je cherche "random ascending sort" dans la doc Array 3.3.5 et je ne trouve rien. »
La doc ne parle pas en intention, elle parle en nom de méthode. Ruby n'appelle pas ça « ascending » : la méthode s'appelle simplement sort, à chercher alphabétiquement à la lettre S. Réflexe à prendre : chercher le NOM probable, pas la DESCRIPTION du besoin.
« sort ou sort! ? »
Le ! signale une mutation : sort! modifie le tableau d'origine et le renvoie. sort laisse l'original intact et renvoie une COPIE triée. Un exercice qui demande « renvoyer une copie triée sans modifier l'original » impose donc sort.
« Pourquoi ma soustraction de dates ne me donne pas un nombre de jours normal ? »
Soustraire deux objets Date renvoie un Rational (une fraction exacte, du type (9876/1)), pas un Integer. Il faut convertir avec .to_i. Et surtout : ne jamais appliquer .to_s à une date avant de la soustraire, car deux chaînes de caractères ne se soustraient pas.
« NameError dans interface.rb alors que la méthode marche. »
Décalage de noms. La méthode définit ses paramètres day, month, year, mais dans l'interface j'avais saisi birth_day, birth_month, birth_year. Une variable locale n'existe que sous le nom sous lequel elle a été créée : Ruby cherche day, ne le trouve pas, et lève un NameError.
« Je répète trois fois puts + gets.chomp.to_i, c'est moche. »
Oui, et c'est exactement le signal d'une factorisation : ce qui varie (le mot « jour », « mois », « année ») devient un paramètre, ce qui se répète devient le corps d'une méthode. D'où ask_birth_info(unit), appelée trois fois avec trois valeurs différentes.

🎯 Concepts clés

Le bang ! : mutation contre copie
Convention Ruby : la version avec ! modifie l'objet receveur en place. Sans !, l'objet d'origine est préservé et on récupère un nouvel objet. Savoir lequel on veut, c'est savoir si on a le droit d'abîmer la donnée d'entrée.
Ruby core ≠ Rails
Une méthode vue dans un tutoriel en ligne peut venir d'ActiveSupport (la bibliothèque de Rails) et non de Ruby. titleize en est le cas typique. Si NoMethodError tombe sur une méthode « qui existe pourtant », la première hypothèse est : elle n'existe pas dans MON environnement.
Le type gouverne les opérations possibles
Une Date se soustrait, une String non. .to_s n'est pas anodin : il change la nature de l'objet, donc l'ensemble des opérations qu'il accepte. Convertir trop tôt, c'est se priver des opérations dont on a besoin.
Factoriser : isoler ce qui varie
Trois blocs identiques à un mot près, c'est une méthode avec un paramètre. La règle de reconnaissance : si je peux décrire les blocs par une phrase unique en remplaçant un mot par un trou, ce trou est mon paramètre.

⚠️ Pièges du jour

Erreur : NoMethodError: undefined method 'titleize' for an instance of String
Cause : titleize appartient à Rails (ActiveSupport), pas à Ruby core. Mon environnement est Ruby seul.
Correctif : AS IS "jean dupont".titleize → TO BE "jean dupont".split.map(&:capitalize).join(" ") # composition de méthodes natives
Erreur : age_in_days renvoie (9876/1) au lieu de 9876.
Cause : la soustraction de deux Date produit un Rational.
Correctif : AS IS Date.today - birth_date → TO BE (Date.today - birth_date).to_i # on convertit la fraction en entier
Erreur : NameError: undefined local variable or method 'day'
Cause : le nom saisi dans l'interface (birth_day) ne correspond pas au nom attendu par l'appel (day).
Correctif : aligner les noms, ou passer explicitement : age_in_days(birth_day, birth_month, birth_year) # l'ordre des arguments compte, pas leur nom
Erreur : le tableau d'origine est modifié alors que l'énoncé demandait une copie.
Cause : usage de sort! au lieu de sort.
Correctif : AS IS an_array.sort! → TO BE an_array.sort # renvoie une copie triée, l'original reste intact

🔧 Patterns & idiomes

require "date"                          # charge la classe Date : indispensable en script autonome

def age_in_days(day, month, year)       # trois paramètres, dans CET ordre
  birth_date = Date.new(year, month, day) # attention : Date.new attend (année, mois, jour)
  today = Date.today                    # la date du jour, en objet Date
  (today - birth_date).to_i             # la soustraction rend un Rational, .to_i le convertit en entier
end                                     # retour implicite de la dernière expression
def ask_birth_info(unit)                # "unit" est ce qui varie : "jour", "mois" ou "année"
  puts "Quel est ton #{unit} de naissance ?" # interpolation : la valeur d'unit s'insère dans la phrase
  gets.chomp.to_i                       # gets lit la saisie, chomp retire le saut de ligne, to_i convertit en entier
end                                     # la méthode renvoie l'entier saisi

day = ask_birth_info("jour")            # un seul appel, réutilisé trois fois
month = ask_birth_info("mois")          # même code, argument différent
year = ask_birth_info("année")          # plus de duplication
"jean dupont".split                     # découpe sur les espaces => ["jean", "dupont"]
             .map(&:capitalize)         # applique capitalize à chaque mot => ["Jean", "Dupont"]
             .join(" ")                 # recolle avec un espace => "Jean Dupont"
