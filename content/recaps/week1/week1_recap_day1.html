🧠 Mes questions & incompréhensions

« Dans ma méthode, je mets puts ou return ? »
Les deux ne servent pas au même public. puts parle à l'humain (il écrit dans le terminal, puis renvoie nil). return parle au programme (il transmet une valeur exploitable par la ligne appelante). Un test RSpec appelle la méthode et regarde sa valeur de retour : il ne lit pas l'écran. Donc dans une méthode testée, c'est return (ou la dernière expression évaluée).
« 3.14, ça suffit pour π ? »
Non, si le test compare à la valeur exacte. Ruby fournit Math::PI, qui est la constante en précision flottante. Écrire une approximation en dur, c'est introduire volontairement une erreur dans un calcul, alors que le langage donne la valeur juste.
« fatal: 'origin' does not appear to be a git repository alors que j'ai bien forké. »
Le fork est une action GitHub : il crée une copie du dépôt sur ton compte, en ligne. Il ne crée aucun lien sur ta machine. C'est le clone qui crée le dossier local et enregistre le remote origin. Diagnostic : git remote -v. S'il ne renvoie rien, aucun remote n'est configuré, donc git push origin n'a aucune adresse où pousser.
« C'est quoi la différence entre origin et upstream ? »
Ce sont deux étiquettes pointant vers deux dépôts distants différents. origin = ton fork (tu y pousses). upstream = le dépôt d'origine du Wagon (tu en tires les mises à jour, tu n'y pousses pas). Rien n'est magique dans ces noms : ce sont des conventions.
« Pourquoi mon mv échoue quand je suis déjà dans level2 ? »
Parce que la destination est exprimée depuis ta position courante. Si tu es dans level2 et que tu veux envoyer un dossier dans level1 (le parent), la destination s'écrit .. et non level1 : depuis l'intérieur de level2, il n'existe aucun chemin nommé level1.
« ~/tmp et /tmp, c'est pareil ? »
Non. /tmp est un chemin absolu : il part de la racine du disque. ~/tmp est relatif à ton dossier personnel, donc /Users/<toi>/tmp. Deux dossiers différents, à deux endroits différents.

🎯 Concepts clés

Valeur de retour d'une méthode
Une méthode est une machine qui rend un objet. Ce qu'elle rend, c'est ce que la suite du programme pourra utiliser. Afficher n'est pas rendre : c'est un effet de bord.
Le conditionnel garde-fou
Tester le cas invalide (rayon négatif) avant de calculer, c'est protéger le calcul. On ne calcule que sur des entrées dont on a vérifié qu'elles ont un sens.
Fork ≠ clone
Le fork vit sur GitHub (serveur à serveur). Le clone vit sur ta machine (serveur vers local) et c'est lui qui crée le lien remote. Tant que tu n'as pas cloné, ton disque ne connaît personne.
Chemin relatif / chemin absolu
Un chemin relatif dépend d'où tu te trouves : il change de sens selon le dossier courant. Un chemin absolu est vrai partout. La plupart des erreurs de terminal viennent d'un chemin relatif interprété depuis la mauvaise position.

⚠️ Pièges du jour

Erreur : la méthode affiche le bon résultat dans le terminal mais le test échoue.
Cause : puts à la place de return. La méthode renvoie nil.
Correctif : AS IS puts Math::PI * radius ** 2 → TO BE return Math::PI * radius ** 2 # la valeur remonte à l'appelant, pas à l'écran
Erreur : git push origin master renvoie fatal: 'origin' does not appear to be a git repository.
Cause : aucun remote enregistré (git remote -v vide), le dossier n'a pas été obtenu par un clone.
Correctif : cloner le fork, ou ajouter le remote explicitement avec git remote add origin <url>.
Erreur : mv configuration level1 échoue depuis level2.
Cause : level1 n'existe pas à l'intérieur de level2, c'est son parent.
Correctif : mv configuration .. # .. désigne le dossier parent

🔧 Patterns & idiomes

def circle_area(radius)                 # on définit une méthode qui reçoit un paramètre "radius"
  if radius < 0                         # on traite d'abord le cas invalide : un rayon négatif n'a pas de sens
    return 0                            # on sort immédiatement de la méthode en renvoyant 0
  end                                   # on ferme le if

  Math::PI * radius ** 2                # dernière expression évaluée : Ruby la renvoie automatiquement
end                                     # on ferme la méthode
git remote -v                           # liste les dépôts distants connus (vide = aucun remote)
git clone <url-de-mon-fork>             # crée le dossier local ET enregistre "origin" automatiquement
git remote add upstream <url-lewagon>   # ajoute le dépôt d'origine sous l'étiquette "upstream"
pwd                                     # affiche le chemin absolu du dossier courant : d'où je parle ?
mv configuration ..                     # déplace "configuration" vers le dossier parent (..)
